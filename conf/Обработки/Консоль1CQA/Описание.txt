В QA выполняется только проверка синтаксиса (set parseonly on). Это позволяет проверять синтаксис запросов,
использующих не существующие (на момент проверки) объекты. Соответственно, при этом пропускаются
возможные ошибки в названиях объектов.
В 1CQA выполняется проверка синтаксиса и компиляция (set noexec on), что позволяет проверить существование объектов.

+ разделение на батчи командой "GO". Разделитель должен быть единственным в строке. План выполнения в окне сообщений
   выводится для всех батчей. Выбор результата выполнения конкретного батча - в правом верхнем углу формы.
+ выводить текст - Включается отладка (в окно сообщений будет выведен пропарсеный текст запроса);
+ RPC mode - ODBCRecordset будет переведен в RPC режим;
+ Старый метод выполнения - выполнение через Открыть() + ПолучитьРезультатыВ_ТЗ(), при этом время выполнения
   отображается как <ВремяОткрытияКурсора> / <ВремяПолученияРезультатов>; если флажок не установлен, то выполнение
   идет методом ВыполнитьИнструкцию(), при этом время отображается как <ВремяОткрытия_Получения>.

   - признак монопольного режима;
   - Каталог базы;

   - Имя пользователя;

  (к сожалению ч/з ODBC выполнить соединение с базой DBF почему-то не удается). 
+ добавлены новые типы параметров запроса:
   - Бухгалтерские счета;
   - Виды субконто (только при выборе в списках ссылок, см.ниже);
   - Планы счетов  (только при выборе в списках ссылок);
   - Календари     (только при выборе в списках ссылок);
   - Виды расчетов (только при выборе в списках ссылок);
+ добавлена возможность укладывать списки ссылок на объекты во временные таблицы (в том числе с иерархическим включением элементов).
  для этого добавлены служебные типы параметров запроса:
   - 'СписокСсылок9';
   - 'СписокСсылок13';
   - 'ИерархияСсылок9';
   параметры этих типов хранят в таблице списки ссылок на объекты определенного типа. 
   для редакирования этих списков в конфигурации или в каталоге обработки ищется обработка с именем "ПодборUChoice".
   если такая обработка есть - она открывается в модальном режиме для редактирования списка ссылок,
   аналогично тому, как это делается обработкой UChoice.ert (УОПО) с диска ИТС (стандартную обработку 
   UChoice приходится приспосабливать, так как в модальном режиме не работают подборы объектов).
   Значения этих параметров (списки ссылок на объекты) перед выполнением запроса укладываются во временные таблицы
   с именами '#<ИмяПараметра>' с помощью методов УложитьСписокОбъектов(), УложитьСписокОбъектов13().
