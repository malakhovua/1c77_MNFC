Настоящая обработка предназначена для выполнения определен-
ных действий над группой документов (операций), отобранных по 
некоторым признакам, а также для формирования реестра докумен-
тов. Вызов обработки осуществоляется из пункта меню «Сервис» 
главного меню программы.    

В форме обработки устанавливается интервал, в котором будут 
выполняться действия над документами и операциями. Период зада-
ется выбором даты начала периода и даты окончания периода.    

Совет. Если нужна обработка за «стандартный» период (месяц, 
квартал, полугодие, 9 месяцев или год), воспользуйтесь кнопкой 
выбора периода  . 

Для выбора видов документов, над которыми будут выполняться 
определнные действия, в форме предназначены два списка. Справа 
выдаются наименования всех документов, существующих в конфи-
гурации. В списке «Виды обрабатываемых документов», располо-
женном слева, указываются виды документов, над которыми предпо-
лагается произвести определнные действия. Переносить документы 
из одного списка в другой можно с помощью двойного щелчка мыши 
или с помощью кнопок «<-» и «->». Для переноса сразу всех видов 
документов существуют кнопки «<<=» и «=>>».  

Выборку документов можно ограничить, указав строку-фильтр по 
комментарию документа или содержанию операции. В этом случае 
обрабатываться будут только документы (операции), у которых 
указанная строка входит в комментарий документа (или содержание 
операции). Если строка-фильтр по содержанию операции указана, 
то документы, не имеющие бухгалтерской операции, обрабатываться 
не будут.
Кроме этого, можно задать фильтр по фирме, по которой создан 
документ, и контрагенту в документе. Если задан фильтр по контрагенту, 
тогда обрабатываются только те документы, в которых есть реквизит 
Контрагент и он совпадает с указанным в качестве фильтра

Дополнительно в реквизите «Обрабатывать документы» можно 
установить признак обработки только проведенных/непроведенных, 
помеченных/непомеченных на удаление или же всех документов. 
Отобрав документы с помощью реквизитов, перечисленных выше, 
можно либо сформировать по ним реестр, либо выполнить над ними 
одно из стандартных действий.     

Для проверки правильности установленных параметров перед 
выполнением обработки рекомендуется сформировать реестр доку-
ментов, воспользовавшись кнопкой "Сформировать реестр".  

Реестр будет содержать список документов за указанный период в 
соответствии с установленными условиями. Для каждого документа 
будет выведена дата, вид документа, номер документа, общая сумма 
документа, а также дополнительная информация, характеризующая 
документ. Дополнительная информация специфична для документов 
различных видов. Порядок записей в реестре соответствует хроноло-
гическому порядку документов.  

Для выполнения групповой обработки документов нужно вы-
брать необходимое действие из списка (в реквизит «Обработка»). 
Действием может быть:
- выключение/включение операции по документу;
- проведение (перепроведение)/отмена проведения документа;
- пометка/отмена пометки на удаление.    

Включение/выключение операций. В некоторых слу-
чаях возникает необходимость временно не учитывать в подсчете 
бухгалтерских итогов суммы отдельных проводок. Для этого преду-
смотрен режим выключения проводок операций. Для повторного 
включения предусмотрен режим включения проводок операций. Для 
выполнения обработки по выключению проводок операций следует 
установить необходимые параметры условий выборки документов, 
выбрать в поле «Обработка» вариант «Выключить проводки» и на-
жать кнопку "Выполнить".     
Для повторного включения в поле «Обработка» следует указать 
вариант «Включить проводки».

Пометка документов, операций/отмена пометки.  При необхо-
димости отдельные документы и операции можно помечать на уда-
ление. В журналах документов и операций это можно сделать только 
для каждого документа (операции) в отдельности. Обработка позво-
ляет пометить на удаление сразу группу документов. Для этого сле-
дует установить необходимые параметры условий выборки докумен-
тов, выбрать в поле «Обработка» вариант «Пометить на удаление» и 
нажать кнопку "Выполнить".

Если в поле «Обрабатывать документы» установлено значение 
«Не помеченные на удаление», то документы в выборке, уже поме-
ченные на удаление, при обработке будут пропускаться.
Для отмены пометки на удаление в поле «Обработка» следует 
указать «Отменить пометку на удаление».    

Если в поле «Обрабатывать документы» установлено значение 
«Помеченные на удаление», то уже документы в выборке, не имею-
щие пометку на удаление, при обработке будут пропускаться.
Обращаем внимание, что непосредственное удаление помеченных 
на удаление объектов из информационной базы производится в 
пункте «Удаление помеченных объектов» меню «Операция» главно-
го меню.
Проведение документов/отмена проведения. Для выполнения 
проведения или отмены проведения группы документов следует ус-
тановить необходимые параметры условий выборки документов, вы-
брать в поле «Обработка» вариант «Провести» или «Сделать не про-
веденными» и нажать кнопку "Выполнить".  

Обработка с вариантом «Провести» выполняет последовательное 
проведение документов в выбранном интервале в соответствии с ус-
тановленными условиями выборки.
Если в поле «Обрабатывать документы» установлено значение 
«Не проведенные», то ранее проведенные документы в выборке бу-
дут пропускаться.
 
Если в поле «Обрабатывать документы» установлено значение 
«Проведенные», то ранее проведенные документы в выборке будут 
перепроводиться.
Обработка с вариантом «Сделать не проведенными» выполняет 
отмену проведения ранее проведенных документов, удовлетворяю-
щих условиям выборки.
Если в поле «Обрабатывать документы» установлено значение 
«Проведенные», то непроведенные документы в выборке будут про-
пускаться.    

Для завершения работы в режиме «Обработка документов» сле-
дует нажать кнопку "Закрыть" или закрыть окно стандартным спо-
собом.

Включение флажка "поддерживать актуальность бух. итогов" на закладке
"Настройки" позволяет несколько ускорить проведение документов.
Наибольший эффект может быть достигнут при массовом проведении
документов, в модулях которых происходит временный расчет бухгалтерских
итогов или выполнение бухгалтерских запросов, например, "Начисление
амортизации", "Финансовые результаты" и т.д. С другой стороны, большинство
документов конфигурации, в том числе "Расходная накладная", "Приходная
накладная (запасы)", "Выпуск продукции", "Оказание услуг" и т.д., выполняют
достаточно простые запросы или не обращаются к бухитогам вообще. Поддержка
актуальности итогов будет замедлять проведение таких документов.
Замечание: этот флажок доступен только в монопольном режиме работы.

Флажок "выполнять обработку в транзакции" позволяет существенно (на один-два
порядка) ускорить выполнение обработки (кроме печати реестра). При включенном
флажке все действия обработки выполняются внутри транзакции, что исключает
частое обращение к относительно медленным внешним носителям данных (жесткий
диск, сетевой диск и т.д.). При этом промежуточные результаты обработки
"удерживаются" в оперативной памяти. В случае неудачи проведения одного
из  документов или любой другой системной ошибке во  время  выполнения
транзакция будет отменена, а информационная база останется в первоначальном
состоянии (до выполнения обработки). Внимание! Использование транзакции в
разделенном (не монопольном) режиме не рекоммендуется. Это может привести
к блокировке работы других пользователей в системе.
