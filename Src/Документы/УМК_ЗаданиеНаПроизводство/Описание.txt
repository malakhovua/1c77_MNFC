Для оформления операций, связанных с отпуском товаров и от-
грузкой продукции собственного производства предназначен доку-
мент "Расходная накладная".

Ввод нового документа осуществляется выбором пункта "Расход-
ная накладная" меню "Документы" / "Расход" главного меню про-
граммы.

Заполнение экранной формы документа следует начать с выбора 
покупателя из справочника "Контрагенты", договора и документа-
счета по которым будут проводиться расчеты. 

Если клиенту уже был выписан "Счет", то "Расходную наклад-
ную", соответствующую отгрузке по данному счету, нужно выписы-
вать только в режиме "Ввод на основании" по этому счету. В этом 
случае договор и документ-счет будут определены автоматически. 

Если "Расходная накладная" является первым документом в це-
почке взаиморасчетов с контрагентом, то при оперативном проведе-
нии расчетов с контрагентом, она будет выступать и в качестве "сче-
та" и в качестве документа долга.

В реквизите "Валюта" следует указать валюту, в которой будет 
выписана накладная (по умолчанию подставляется валюта договора) 
и курс этой валюты. Отметим, что выбирать валюту, отличную от 
национальной валюты, можно только для контрагентов-
нерезидентов.

Реквизит "Вид торговли" установится в то значение, которое ука-
зано по умолчанию для данного клиента в справочнике "Контраген-
ты". При этом, если выбран вид торговли "за наличные", то допол-
нительно в реквизите "Касса" нужно указать кассу, в которую по-
ступает выручка. 

Обратим внимание на то, что документ допускает продажу за ва-
люту только контрагентам-нерезидентам и только на условиях пред-
оплаты.
 
Далее выбирается склад, с которого отгружается продукция, това-
ры, тип цен и вид тары, в которой отгружаются ТМЦ. 

В табличной части "Расходной накладной" указывается:
	- наименование ТМЦ;
	- предпочтение при списании партий ТМЦ;
	- количество единиц отгружаемого товара (продукции);
	- единица измерения;
	- коэффициент пересчета для выбранной единицы измере-
ния;
	- цена ТМЦ в выбранной валюте без НДС и (или) цена с 
НДС (в зависимости от значения константы "Цены в на-
кладных");
	- сумма без НДС по строке (без скидки);
	- сумма скидки;
	- суммы по строке без НДС и с НДС с учетом скидки;
	- сумма налога на добавленную стоимость. При этом, если в 
шапке документа указана ставка 20% (реквизит "Вид 
НДС" на закладке "Дополнительно"), то для каждой по-
зиции в табличной части будет выбираться та ставка 
НДС, которая указана для нее в справочнике "Номенкла-
тура". В противном случае - в шапке выбрано другое 
значение ставки НДС - именно с учетом этой ставки бу-
дет рассчитываться сумма НДС по каждой позиции таб-
личной части.

В графе "Полн. своб. остаток" отображается остаток по текущей 
позиции по всей фирме с учетом или без учета резерва (в зависимо-
сти от значения константы "Разрешить продавать резерв"), в графе 
"Ост. по складу" - остаток текущего ТМЦ на конкретном складе 
всегда без учета резерва. 

Отметим, что реквизиты "Сумма б/с", "Скидка" появляются в 
табличной части документа только в случае, если константа "Ис-
пользовать скидку" установлена в значение "Да". В этом случае и 
итоговые суммы по строке с НДС и без НДС будут рассчитаны с 
учетом скидки. 

С помощью кнопки "Дать скидку" можно задать различные спо-
собы расчета скидок: заданным процентом или суммой для каждого 
товара в отдельности "(опции "Суммой по строке", "Процентом по 
строке"), или для всей накладной (после ее заполнения) в целом 
(опции "Суммой по накладной", "Процентом по накладной").
При введении на основании "Расходной накладной" "Налоговой 
накладной" сумма скидки будет подставлена в соответствующую 
графу этого документа.

Заполнение табличной части документа можно производить не-
сколькими способами: обычным построчным вводом или методом 
множественного подбора из справочников "Номенклатура" или 
"Прайс лист". Для этого в диалоговой форме документа присутству-
ет кнопка "Подбор".

На закладке "Дополнительно" можно внести информацию о до-
веренности, на основании которой были отпущены ТМЦ  (реквизи-
ты в группе "Доверенность") и о сотруднике, который отпустил то-
вары (реквизит "Отпустил"). Напомним, что сотрудник, который 
обычно отпускает товары (продукцию) может быть задан в констан-
те "Кто обычно отпускает товары". Здесь также можно указать 
подразделение сбыта, для отражения аналитики по доходам, и 
если в табличной части встречаются калькулируемые услуги,
производственное подразделение -- для аналитики по затратам
производства таких услуг.

В реквизитах "Субконто валовых доходов", "Вид НДС", "Под-
разделение" указывается аналитика для бухгалтерских проводок по 
валовым доходам, НДС, проводке по списанию себестоимости про-
данных ТМЦ. 

С помощью кнопки "Печать" можно получить печатную форму 
документа.

Заполненную форму накладной необходимо сохранить и провести 
(кнопка "ОК").

При проведении документа автоматически формируются бухгал-
терские проводки, состав которых зависит от типа номенклатуры, 
вида торговли и состояния взаиморасчетов с контрагентом по дан-
ным оперативного учета. При этом, если в конфигурации включен 
режим автоматического выделения и погашения выданных и полу-
ченных авансов, то при проведении документа будет сформирована 
проводка по зачету выплаченного покупателем аванса (предоплаты). 
В оперативном учете фиксируется списание партий ТМЦ, изменение 
количественного остатка по всем ТМЦ табличной части (в регистрах 
"Партии" и "Остатки" соответственно), изменение состояния взаи-
морасчетов с покупателем (регистр "Взаиморасчеты с поставщика-
ми"), увеличение оборотов (в регистре "Обороты").
