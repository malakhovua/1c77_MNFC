В конфигурации предусмотрена возможность комплексного 
формирования цен на группу выбранных товаров. В процессе 
формирования можно проконтролировать цены с помощью 
специального отчета, который вычисляет наценку текущей цены к 
себестоимости товара или к его учетной цене (в процентах). Процесс 
формирования цен можно разбить на несколько этапов: установка 
фильтра на список позиций номенклатуры, контроль цен с помощью 
специального отчета, формирование новых цен или изменение 
существующих. 


УСТАНОВКА ФИЛЬТРА НА СПИСОК ЦЕН ПОЗИЦИЙ НОМЕНКЛАТУРЫ

В окне настройки требуется задать фильтры на элементы 
справочника цен, подчиненного справочнику товаров, 
чтобы отобрать из них подлежащие групповой 
обработке. Окно настройки имеет две закладки: 
"Основная" и "Множественный фильтр". 

После установки параметров фильтра у пользователя имеется две 
возможности - проконтролировать список обрабатываемых позиций 
(кнопка "Просмотр" или "ОК") и сформировать новые цены или 
изменить существующие (кнопка "Выполнить").

Все параметры фильтра (указанные на основной закладке и во 
множественном фильтре) действуют в совокупности. Поэтому к 
установке параметров в основном и множественном фильтре надо 
подходить обдуманно. 

Например, если в основном экране задан какой-то 
конкретный тип цен, а во множественном фильтре - 
список типов цен, в которую этот тип цен не входит, 
то после задания такого фильтра по кнопке "Выполнить"
будет выдан список номенклатуры с себестоимостью и 
учетной ценой, но без цен какого-либо типа. По кнопке
"Просмотр" или "ОК" будет получена пустая таблица.

Совет! Перед тем, как установить параметры одновременно в двух 
фильтрах, надо хорошо обдумать, какие именно данные 
желательно получить. Если задачу четко сформулировать не 
удается, то лучше пользоваться либо тем, либо другим 
фильтром. 

В случае если множественный фильтр установлен, перед 
названием закладки "Множественный фильтр" устанавливается знак 
(!). Он виден и с основной закладки.

В окне подготовки можно установить следующие фильтры:

Тип цен - задать конкретный тип цен, который установлен для 
позиций номенклатуры. С помощью множественного 
фильтра можно задать список типов цен. В 
последнем случае будут отобраны все позиции, у 
которых имеется хотя бы один тип цен из списка.

Показывать себестоимость за - выбрать из выпадающего 
списка, за какую единицу измерения показывать 
себестоимость товара при контроле и изменении цен: 
за единицу, установленную по умолчанию для товара, 
или за базовую единицу. 

Фильтр на позиции номенклатуры:

Вид - выбрать из выпадающего списка вид товарной 
номенклатуры: вся номенклатура, только товары, 
только услуги, только наборы. 

Номенклатура - задать конкретную позицию номенклатуры или 
группу позиций номенклатуры. С помощью 
множественного фильтра можно задать список 
произвольно выбранных позиций или групп позиций.

Категория - задать конкретную категорию позиции 
номенклатуры. С помощью множественного фильтра 
можно задать список категорий позиций. Причем 
отбор позиций будет произведен по разным 
алгоритмам, в зависимости от установленного 
переключателя: одна из или одновременно все. В 
первом случае будут отобраны позиции, которые 
относятся хотя бы к одной из указанных в списке 
категорий, во втором - только позиции, относящиеся 
сразу ко всем категориям, указанным в списке.

Только имеющиеся на складах - установка этого 
переключателя означает, что отбираются для 
обработки только те позиции номенклатуры, которые 
есть в наличии на каком-нибудь складе компании. 

Только отсутствующие на складах - установка этого 
переключателя означает, что отбираются для 
обработки только те позиции номенклатуры, которых 
нет в наличии ни на каком складе компании.

Фильтр на цены:

Валюта - задать конкретную валюту цены. Отбираются те 
позиции, цены для которых имеют заданную валюту. 
С помощью множественного фильтра можно задать 
список валют. В этом случае отбираются те позиции, 
у которых валюта цены совпадает хотя бы с одной из 
валют списка. 

Единица измерения - задать конкретную единицу измерения 
количества товара. Отбираются те позиции, у 
которых имеется цена за данную единицу измерения. 
С помощью множественного фильтра можно задать 
список единиц измерения, по которым будет 
производиться отбор позиций. В этом случае 
отбираются те позиции, у которых имеется цена за 
какую-либо из заданных в списке единиц измерения. 

Цена - установить интервальный фильтр по цене. 

Наценка на себестоимость (%) - установить интервальный 
фильтр по наценке на себестоимость позиции в 
процентах. В этом случае будут отобраны те позиции, 
для которых в заданном интервале находится 
значение наценки, вычисленное по значениям цены и 
текущей себестоимости. 

Наценка на учетную цену (%) - установить интервальный 
фильтр по наценке на учетную цену позиции в 
процентах. В этом случае будут отобраны те позиции, 
для которых в заданном интервале находится 
значение наценки, вычисленное по значениям цены и 
текущей учетной цены. 

Фильтр сравнения: Наценка на себестоимость или Наценка на 
учетную цену сравнивается (больше, меньше, равна 
и др. условия) с наценкой в справочнике на 
отбираемую цену. Отбираются позиции, у которых 
наценка удовлетворяет установленному условию. 

ВНИМАНИЕ!!! Все параметры фильтра действуют в совокупности. То 
есть, если хотя бы один из параметров не удовлетворяет 
заданным условиям, то эта позиция отобрана не будет. 
Например, если в фильтре установлен тип цены - "розничная", 
группа - "Инвентарь", интервал цен "от 100 USD", а все 
товары группы "Инвентарь" имеют розничную цену ниже 
100 USD, то они не будут отобраны для дальнейшей обработки.

ВНИМАНИЕ!!! Для того чтобы получить интервальный фильтр с 
открытой границей интервала, нужно установить (или оставить) 
соответствующую границу равной нулю. Например, если задать
фильтр по ставке наценки на себестоимость "от 10 % до 0",
заданы на самом деле окажутся границы "от 10 % до любой величины".
