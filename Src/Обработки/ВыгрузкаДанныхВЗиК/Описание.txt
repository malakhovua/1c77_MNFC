Для вызова формы Помощника нужно выбрать одноименный 
пункт меню "Сервис" / "Помощники" главного меню программы.

На первой странице Помощника содержится приветственное со-
общение и приглашение к началу работы по выгрузке данных. Си-
ним шрифтом на каждой странице Помощника отображается пояс-
няющий текст по каждому его реквизиту и управляющему элементу, 
черным шрифтом в нижней части страницы - описание тех действий, 
которые необходимо выполнить.

На второй странице Помощник предлагает восстановить сохра-
ненные настройки, которые были установлены во время предыдуще-
го сеанса работы с ним. Настройка представляет собой список сче-
тов, которые были введены с целью их последующей загрузки в 
справочник "План счетов" конфигурации "Зарплата + Кадры для 
Украины" (см. ниже), а также фильтры для отбора объектов анали-
тики этих счетов. Список предлагаемых настроек всегда содержит 
как минимум один элемент "по умолчанию", который и будет пред-
ложен при первом сеансе работы с Помощником. 

На следующей странице Помощника нужно отметить те счета из 
предлагаемого списка, которые должны быть помещены в справоч-
ник "План счетов" конфигурации "Зарплата + Кадры для Украи-
ны". По умолчанию здесь предложен список всех субсчетов тех сче-
тов, которые согласно корректным проводкам могут корреспондиро-
вать со счетом 66 "Расчеты по оплате труда".
С помощью опции "Представлять код счета по шаблону ##.#.#.#"
можно осуществлять выгрузку счетов с использованием 
точек-разделителей (флажок включен), либо без их использования 
(флажок выключен). Для корректного взаимодействия с планом счетом 
в конфигурации "Зарплата + Кадры для Украины" рекомедуется 
использовать первый вариант (флажок включен).
 
На следующей странице Помощника вы по сути определяете со-
держимое справочников "Виды субконто" и "Объекты аналитиче-
ского учета" конфигурации "Зарплата + Кадры для Украины". В 
верхней ее части перечислены все виды субконто счетов, выбранных 
на предыдущей странице. В конце этого списка присутствует вид 
субконто, который будет использован в качестве третьего вида суб-
конто на счете 23 в конфигурации "Зарплата + Кадры для Украи-
ны". Тип этого субконто определяется значением константы текущей 
конфигурации "Тип дополнительной аналитики для конфигурации 
"Зарплата+Кадры для Украины". В качестве его аналитики могут 
быть выбраны виды продукции (справочник "Номенклатура") или 
заказы на ее производство (документы "Заказы"). В обоих случаях 
выбранный тип объекта будет фигурировать в качестве "заказа" в 
конфигурации "Зарплата + Кадры для Украины". Так, если на пред-
приятии ведется массовое производство продукции, то учет затрат на 
заработную плату целесообразно вести в разрезе наименований про-
дукции: т.е. в качестве дополнительной аналитики нужно выбрать 
"продукцию". Тогда в качестве понятия "заказ" как он используется 
в конфигурации "Зарплата + Кадры для Украины" будет фигуриро-
вать продукция и в соответствующих документа (нарядах, докумен-
тах распределения основных начислений) в реквизите "заказ" будет 
предлагаться выбор из загруженных элементов справочника "Но-
менклатура" текущей конфигурации.

Аналогично, если на предприятии используется преимущественно 
позаказная схема производства продукции, в качестве дополнитель-
ной аналитики имеет смысл выбрать объект "заказ" (субконто, 
имеющий тип "документ"). Тогда в конфигурации "Зарплата + Кад-
ры для Украины" в качестве понятия "заказ" будет фигурировать 
заказ, как он понимается в текущей конфигурации - т.е. документ, 
по которому оформлен заказ на производство продукции.
 
Позиционируя курсор на конкретном виде субконто, можно уста-
новить фильтр, по которому будет происходить отбор объектов тако-
го вида. Так, в качестве фильтра для субконто-справочника, можно 
задать элемент или группу его элементов, которые будут помещены в 
справочник "Объекты аналитического учета". Форма соответствую-
щего справочника открывается по кнопкам "…", "....". Для субконто-
документов в качестве фильтра можно задать дату. В этом случае в 
справочник будут выгружены только документы, по которым есть 
ненулевые остатки, а также документы, введенные позже этой даты.

Если фильтр для какого-либо вида субконто не задан, то в спра-
вочник "Объекты аналитического учета" для него будут выгружены 
все объекты: т.е. все элементы соответствующего справочника или 
все документы.

При первом сеансе работы с Помощником рекомендуется выпол-
нить также выгрузку схемы хозяйственных операций и соответст-
вующих им проводок, предлагаемых по умолчанию, т.е. по сути за-
полнить справочники "Хозяйственные операции" и "Проводки" 
конфигурации "Зарплата + Кадры для Украины". Чтобы иницииро-
вать эту операцию нужно включить соответствующий флажок на 
следующей странице Помощника.
 
	ВНИМАНИЕ! Выгрузку схемы проводок следует производить только 
	один раз при первом сеансе работы с Помощником. В против-
	ном случае при последующих сеансах работы с Помощником 
	все существующие хозяйственные операции (в одноименном 
	справочнике конфигурации "Зарплата + Кадры для Украины") 
	будут помечены на удаление, что приведет к некорректному 
	экспорту проводок.

Рассмотрим подробнее порядок настройки параметров такой опе-
рации.

В предлагаемой по умолчанию схеме проводок для хозяйствен-
ных операций по всем начислениям, суммы которых должны отра-
жаться по дебету затратных счетов, не определены счета затрат и 
объекты их аналитики: вид деятельности и вид затрат. Поэтому на 
следующих страницах Помощника вам предлагается выбрать счет 
затрат и объекты его аналитики для проводок таких операций. Если 
в предлагаемом списке затратных счетов вы отметите больше, чем 
один счет, то по каждой хозяйственной операции будет сформирова-
но несколько операций с проводками, отличающимися выбранными 
вами счетами.
 
Аналитику "вид деятельности" можно оставить "пустой", или же 
выбрать конкретные виды деятельности из предлагаемого списка. 
Если в списке вы отметите больше, чем один вид деятельности, то по 
каждой хозяйственной операции будет сформировано несколько 
операций. Проводки по таким операциям будут отличаться выбран-
ными видами деятельности.
 
Наконец, последней настройкой для начальных проводок являет-
ся определение вида затрат для таких операций, где указанная ана-
литика не определена. Аналогично, если в режиме "из списка" вы 
отметите больше, чем один вид затрат, то для каждой такой опера-
ции будет сформировано несколько хозяйственных  операций с про-
водками, отличающимися выбранными видами затрат.
 
На следующей странице Помощник предлагает сохранить все вы-
бранные вами параметры в виде отдельной настройки.

При сохранении настройки под именем уже существующей, все 
параметры, которые были в этой настройке будут заменены на но-
вые. Если сохранить настройку под новым именем (флажок "Сохра-
нить под новым именем"), при следующем сеансе работы с Помощ-
ником она будет представлена в списке пользовательских настроек.

Последним действием перед формированием файла выгрузки яв-
ляется задание имени такого файла и собственно инициирование 
самого процесса выгрузки (с помощью кнопки "Дальше").

Формирование файла выгрузки данных может занять длительное 
время в зависимости от объема информационной базе. При этом все 
выполняемые в текущий момент действия будут представлены в 
специальном окне "Процесс выполнения".

По кнопке "Просмотр" на последней странице Помощника загру-
жаемый файл открывается в окне тестового редактора.

Завершение работы с Помощником выполняется по кнопке 
"Готово".

После того, как файл выгрузки сформирован, нужно импортиро-
вать выгруженные данные в конфигурацию "Зарплата +Кадры для 
Украины". Для этого необходимо воспользоваться стандартной об-
работкой "Загрузка данных из текстового файла", которую нужно 
запустить в конфигурации "Зарплата + Кадры для Украины", вы-
брав в ней в качестве файла загрузки только что сформированный 
файл. В результате ее выполнения в конфигурации "Зарплата + 
Кадры для Украины" будут заполнены справочники: "План счетов", 
"Виды субконто", "Объекты аналитического учета" и "Хозяйствен-
ные операции" (только при первом его выполнении).
